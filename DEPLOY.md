# Деплой на Timeweb App Platform

Приложение готово к деплою из Git-репозитория. Зависимостей нет, сборка идёт по `Dockerfile`.

## Ключевые настройки в App Platform
- **Тип приложения:** Backend / Docker (сборка из `Dockerfile`).
- **Порт:** `4000` (приложение слушает `process.env.PORT`, по умолчанию 4000).
- **Команда запуска:** не нужна — задана в `Dockerfile` (`node server.js`).
- **Переменные окружения:** не требуются (при желании можно задать `PORT`).
- **Node:** 24 (зафиксирован в `Dockerfile`, нужен для встроенного `node:sqlite`).

## Источник кода
App Platform разворачивает приложение из Git. Варианты:
1. **GitHub / GitLab** — подключаются через OAuth в браузере, затем выбирается репозиторий.
2. **Встроенный Git Timeweb** — Timeweb выдаёт git-адрес, в который пушится код.

## Важно: хранение данных
Файловая система App Platform **временная** — база `srm.db` пересоздаётся с демо-данными
при каждом передеплое/перезапуске. Для постоянного хранения нужно либо подключить
постоянный диск (если доступен в тарифе), либо перенести данные в управляемую БД (PostgreSQL).
