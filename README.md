# СИТИ SRM — серверная версия (Этап 2)

Многопользовательская версия SRM-системы: бэкенд с базой данных и API, аутентификация,
роли и права доступа, управление сотрудниками, постановка и контроль задач с напоминаниями о сроках.

В отличие от прототипа Этапа 1 (`../srm-sistema.html`, один файл на localStorage),
здесь **данные общие для всех пользователей** и хранятся на сервере в базе SQLite.

## Запуск

Требуется **Node.js ≥ 22.5** (используется встроенный `node:sqlite`). Внешних зависимостей нет —
`npm install` не нужен.

```bash
cd server-app
node server.js
```

Откройте **http://localhost:4000** в браузере. Порт можно сменить: `PORT=8080 node server.js`.

База данных создаётся автоматически в файле `server-app/srm.db` и наполняется демо-данными
при первом запуске.

## Возможности

- **Регистрация и вход** — экран входа/регистрации, выбор роли (прав доступа), сессия в httpOnly-cookie (7 дней).
- **Права доступа (RBAC)** — 6 ролей; модули скрываются и кнопки редактирования отключаются по правам;
  права проверяются и на сервере (нельзя обойти через API). Матрица прав видна на странице «Сотрудники».
- **Сотрудники** — реестр пользователей: добавление, смена роли/должности, отключение, удаление (только администратор).
- **Задачи** — постановка задач сотрудникам с указанием помещения, срока и приоритета;
  канбан-доска (Открыто → В работе → Готово), отслеживание выполнения, счётчик активных и просроченных задач у каждого сотрудника.
- **Напоминания о сроках** — колокольчик в шапке с количеством; список задач, просроченных или со сроком ≤ 3 дней;
  подсветка карточек (красная — просрочено, жёлтая — скоро).
- **Несколько объектов (портфель)** — управление зданиями по разным адресам; переключатель объекта в меню («Все объекты» или конкретный); все модули (дашборд, занятость, договоры, платежи, коммуналка, отчёты) фильтруются по выбранному объекту, метрики агрегируются по портфелю.
- **Многопользовательский режим** — общая БД; лёгкий автоопрос (30 с) подтягивает изменения других пользователей.
- **Все модули Этапа 1** — дашборд, объекты и занятость, арендаторы, договоры, платежи, коммуналка и расходы, отчёты (NOI, экспорт CSV).
- **Тёмная/светлая тема**, фирменный логотип.

## Архитектура

```
server-app/
  server.js        — HTTP-сервер, маршрутизация, аутентификация, API, раздача статики
  db.js            — схема SQLite, роли/права, демо-данные, хэширование паролей (scrypt)
  package.json
  public/
    index.html     — каркас, экран входа/регистрации, стили
    app.js         — фронтенд: SPA, API-клиент, все модули
  srm.db           — база данных (создаётся автоматически, не входит в репозиторий)
```

Стек: `node:http` + `node:sqlite` + `node:crypto` (без внешних зависимостей).
Пароли — scrypt с солью; токен сессии — HMAC-SHA256 в httpOnly-cookie.

## Сброс данных

Администратор может сбросить операционные данные к демо кнопкой «↺ Демо-данные» на дашборде
(пользователи сохраняются). Полный сброс — удалить файл `srm.db` и перезапустить сервер.
