# Развёртывание СИТИ SRM на VPS — пошаговая инструкция

Эта инструкция поднимает рабочую систему на отдельном сервере (VPS), где данные хранятся
постоянно. Технические детали спрятаны: достаточно по очереди выполнить команды.

**Что получится:** система открывается в браузере по адресу сервера (а позже — по вашему домену
с HTTPS), данные всех арендаторов, договоров и платежей лежат в базе на диске сервера и
автоматически копируются каждую ночь.

**Где хранятся данные:** файл базы `deploy/data/srm.db` на диске VPS. Он не пропадает при
перезапуске и обновлении системы. Резервные копии — в `deploy/backups/`.

---

## Шаг 0. Отправить новые файлы в GitHub (на вашем Mac)

Я добавил в проект конфигурацию для сервера (папка `server-app/deploy/`). Её нужно один раз
отправить в репозиторий, чтобы сервер мог её скачать. В Терминале на Mac:

```bash
cd "/Users/sergejzajkin/Documents/Claude/Projects/СИТИ srm/server-app"
git add deploy db.js .dockerignore
git commit -m "Конфигурация для развёртывания на VPS (Docker + Caddy + бэкапы)"
git push
```

## Шаг 1. Арендовать VPS

В панели Timeweb → «Облачные серверы» → создать сервер:

- **ОС:** Ubuntu 24.04
- **Конфигурация:** 2 vCPU, 2–4 ГБ RAM, 40 ГБ SSD (минимальный тариф для старта подойдёт)
- **Регион:** Россия (важно для 152-ФЗ — данные граждан РФ должны храниться в РФ)

После создания вы получите **IP-адрес** сервера и **пароль root** (или загрузите свой SSH-ключ).

---

## Шаг 2. Подключиться к серверу

В Терминале на Mac (замените `IP_СЕРВЕРА` на адрес из панели):

```bash
ssh root@IP_СЕРВЕРА
```

Введите пароль. Вы — на сервере. Дальнейшие команды выполняются здесь.

---

## Шаг 3. Установить Docker

Скопируйте и выполните одной командой:

```bash
apt update && apt install -y docker.io docker-compose-plugin sqlite3 git
systemctl enable --now docker
```

Проверка: `docker --version` должна показать версию.

---

## Шаг 4. Скачать проект

```bash
cd /opt
git clone https://github.com/avgur264-bot/citi-srm.git
cd citi-srm/deploy
chmod +x backup.sh deploy.sh
```

---

## Шаг 5. Настроить и запустить

```bash
cp .env.example .env
docker compose up -d --build
```

Первая сборка занимает 1–2 минуты. Проверка, что всё работает:

```bash
docker compose ps        # оба контейнера должны быть "running"
docker compose logs app  # в логах: сервер слушает порт 4000
```

---

## Шаг 6. Открыть систему и сменить пароли

Откройте в браузере: **http://IP_СЕРВЕРА**

Вход под демо-администратором: `admin@citisrm.ru` / `admin123`

**Сразу же** в модуле «Сотрудники»:
1. Смените пароль администратора на свой надёжный.
2. Заведите реальных пользователей с нужными ролями.
3. Удалите неиспользуемые демо-аккаунты (manager, lease, buh, exp, owner).

> Демо-пароли публично известны — оставлять их в боевой системе нельзя.

---

## Шаг 7. Включить файрвол

Откроем только нужные порты (SSH, веб):

```bash
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
```

---

## Шаг 8. Домен и HTTPS (когда будет домен)

1. У регистратора домена создайте **A-запись**, указывающую на IP вашего сервера
   (например, `srm.вашакомпания.ru → IP_СЕРВЕРА`). Подождите, пока обновится (до часа).
2. На сервере впишите домен в `.env`:

   ```bash
   cd /opt/citi-srm/deploy
   nano .env
   ```
   Замените строку на:
   ```
   SITE_ADDRESS=srm.вашакомпания.ru
   ```
   Сохраните (Ctrl+O, Enter, Ctrl+X).
3. Перезапустите:
   ```bash
   docker compose up -d
   ```

Caddy сам выпустит бесплатный SSL-сертификат. Через минуту система откроется по
**https://srm.вашакомпания.ru** (замочек в браузере). Сертификат продлевается автоматически.

---

## Шаг 9. Настроить ежедневные бэкапы

Добавим задачу в планировщик (каждую ночь в 03:00):

```bash
( crontab -l 2>/dev/null; echo "0 3 * * * /opt/citi-srm/deploy/backup.sh >> /var/log/citi-srm-backup.log 2>&1" ) | crontab -
```

Проверить вручную прямо сейчас:

```bash
/opt/citi-srm/deploy/backup.sh
ls -lh /opt/citi-srm/deploy/backups
```

Копии хранятся 14 дней (старые удаляются автоматически).

> Рекомендуется дополнительно скачивать бэкапы на другой носитель/в облако — на случай отказа
> самого сервера.

---

## Обновление системы в будущем

После того как вы внесёте изменения в код и отправите их в GitHub (`git push` с Mac), на сервере:

```bash
/opt/citi-srm/deploy/deploy.sh
```

Скрипт скачает свежий код, пересоберёт и перезапустит систему. **База данных не затрагивается.**

---

## Восстановление из бэкапа

Если нужно вернуть данные из копии:

```bash
cd /opt/citi-srm/deploy
docker compose stop app
cp backups/srm-ВЫБРАННАЯ_ДАТА.db data/srm.db
rm -f data/srm.db-wal data/srm.db-shm
docker compose start app
```

## Частые вопросы

**Сайт не открывается по IP.** Проверьте файрвол (шаг 7) и `docker compose ps`. Логи: `docker compose logs caddy` и `docker compose logs app`.

**HTTPS не выпускается.** Убедитесь, что A-запись домена указывает на сервер, порт 443 открыт, и в `.env` указан домен без `http://`.

**Где лежат данные?** `/opt/citi-srm/deploy/data/srm.db`. Эту папку и нужно беречь/копировать.

**Сколько потянет сервер?** Для одной управляющей компании и десятков пользователей минимального
тарифа достаточно. При росте — увеличить тариф или перейти на отдельную базу PostgreSQL.
