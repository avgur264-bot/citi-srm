# СИТИ SRM — установка на собственный сервер клиента

Система переносимая: Node 24 + встроенный SQLite, без внешних зависимостей.
Вся база — один файл `srm.db` на диске сервера (данные остаются у клиента).

## Что нужно от сервера
- ОС **Linux (Ubuntu 22/24)** — рекомендуется. (Windows Server — через Docker Desktop или запуск Node напрямую.)
- **2 ГБ RAM**, ~10 ГБ диска (SQLite крошечный), 1–2 ядра.
- Доступ к интернету для установки (можно и закрытый контур — см. ниже).
- (По желанию) **доменное имя** для красивого адреса и HTTPS. Без домена — работа по IP.

---

## Вариант 1 — через Docker (рекомендуется, 5 шагов)

```bash
# 1. Установить Docker (один раз)
curl -fsSL https://get.docker.com | sh

# 2. Забрать код
git clone https://github.com/avgur264-bot/citi-srm.git citi-srm
cd citi-srm/deploy

# 3. (по желанию) задать домен для авто-HTTPS
echo "SITE_ADDRESS=srm.компания.ru" > .env      # без домена — пропустить, будет :80 по IP

# 4. Запустить
docker compose up -d --build

# 5. Открыть
#   по IP:    http://IP-СЕРВЕРА
#   по домену: https://srm.компания.ru   (сертификат выпустится сам)
```

**Где база:** `citi-srm/deploy/data/srm.db` — на диске сервера, переживает перезапуски и обновления.

---

## Вариант 2 — без Docker, напрямую на Node 24

```bash
# Нужен Node.js версии 24+ (в нём встроенный SQLite)
cd citi-srm
PORT=80 DB_PATH=/var/lib/citisrm/srm.db node server.js
# для постоянной работы оформить как systemd-сервис
```

---

## Обновления (когда выходит новая версия)
```bash
cd citi-srm/deploy && bash deploy.sh      # git pull + пересборка, база не трогается
```
Для закрытого контура (без интернета) — новую версию передаёт поставщик архивом.

## Резервные копии
```bash
cd citi-srm/deploy && bash backup.sh      # копия базы в ./backups (хранит 14 шт.)
```
Рекомендуется поставить в cron на ежедневный запуск.

## Перенос данных из демо/нашего хостинга
Скопировать файл `srm.db` со старого сервера в `citi-srm/deploy/data/srm.db` и перезапустить
(`docker compose restart`). Все данные переедут.

## Безопасность (перед боевым запуском)
- Сменить демо-пароли (войти под `admin`, в «Сотрудниках» задать новые) или развернуть с
  `SEED_PASSWORD=...` в окружении приложения.
- Регистрация по умолчанию закрыта; новых пользователей заводит админ.
- Для HTTPS указать домен в `.env` (Caddy выпускает бесплатный сертификат автоматически).
- Закрытый контур (внутренняя сеть, без интернета) — поддерживается: доступ по внутреннему IP.

## Кто устанавливает
- **Админ клиента** — по этой инструкции, самостоятельно. ИЛИ
- **Поставщик** — по SSH-доступу к серверу клиента (установка + дальнейшие обновления).
